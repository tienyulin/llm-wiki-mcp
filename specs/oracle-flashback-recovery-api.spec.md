## §0 全域規則（全端點一體適用，端點不得自定）

### 0.1 閘門順序

```
1. auth        → 401（mutation 端點缺/錯 X-API-Key；read 端點不驗）
2. schema 驗證 → 422（pydantic：必填、型別、互斥輸入）
3. 資源解析    → 404（請求指到的資源不存在；dry_run 也會 404）
4. 風險閘門    → 428（irreversible 且 dry_run=false 時檢查 confirm/approval_id；
                     dry_run=true 不檢查）
5. 前置條件    → 409（領域狀態不允許）
6. 執行
dry_run=true：跑 1–3，前置條件以「收集不擲出」模式全部執行，回 checks，不進 4/6
```

### 0.2 統一 response 形狀

```json
// dry_run=true（所有 reversible/irreversible 端點）：
{"dry_run": true,
 "checks": {"<條件名>": {"ok": true}
            或 {"ok": false, "detail": "...", "error_code": "ORA-xxxxx"|null},
 ...端點唯讀附加欄位（各端點 AC 列出）}

// dry_run=false 成功：{"dry_run": false, ...端點專屬欄位}

// 所有錯誤（統一 exception handler）：
{"detail": "human readable, 前置條件違反時以 'Pn violated: ' 開頭", "error_code": "ORA-xxxxx"|null}
```

### 0.2.1 全域型別約定

- 所有時間欄位（`*_time`、`time`、`droptime`、`timestamp`）一律
  **ISO8601 秒精度、無時區（naive UTC）**：`YYYY-MM-DDTHH:MM:SS`
- response 中 bool 欄位永遠出現（不省略 false）
- 「droptime 最近」一律指 **max(droptime)**；排序平手時以插入順序（FIFO）穩定排序

### 0.3 風險閘門常數

- confirm token 字面值：`"I-UNDERSTAND-DATA-LOSS"`——實作放單一常數（models 層），
  service 與測試都引用該常數
- `approval_id`：非空白字串（strip 後長度 > 0）。confirm 與 approval_id 的驗證
  **發生在閘門 4**（service 層，僅 irreversible 且 dry_run=false 時），不是 pydantic
  ——schema 上兩者都是 `Optional[str] = None`

### 0.4 auth 與 DI 模式（內容 inline，不依賴其他檔案）

- auth：`X-API-Key` header 與環境變數 `FLASHBACK_API_KEY` 用 `secrets.compare_digest`
  比對；env **於每個 request 時讀取**（測試可 monkeypatch）；未設定 = dev 模式不驗
- DI：repository/service 為 lazy 單例——`functools.lru_cache(maxsize=1)` provider
  函式 + FastAPI `Depends`；提供 `reset_singletons()` 給測試清除；
  `main.py` lifespan 呼叫一次 service provider 做 warm-up（設定錯誤在 boot 時失敗，
  測試的 TestClient 不跑 lifespan 所以維持 lazy）
- 統一錯誤：service 擲 `FlashbackError(status_code, detail, error_code)`，
  app 級 exception handler 轉 §0.2 錯誤 body
- 環境變數讀取時機：**`FLASHBACK_API_KEY` 與 `FRA_USAGE_THRESHOLD` 每 request 讀**
  （`os.getenv`，不進 Settings 快取）；其餘（`MOCK_ORACLE`、`ORACLE_DSN`…）boot 時
  快取於 Settings。real 模式缺 `ORACLE_DSN` → lifespan warm-up 直接失敗（fail-hard，
  無降級）

### 0.5 同步模型與並發

- 所有端點**同步**執行（單實例管理 API；真實 Flashback Database 可耗時數分鐘，
  client 自行設定 timeout）。202+job 模式列 Out of Scope（§9）
- 並發防護：database 級操作由 §2.5 狀態機擋（非 OPEN 不能 flashback、非 FLASHBACKED
  不能 finalize）；table/restore point 級操作**無並發防護**（last-write-wins），
  風險接受：管理操作預期單人執行
- 冪等性：無冪等 key。同請求重打的行為由各端點 AC 定義（例：重複建 RP → 409）

## §2.5 狀態機（db_state）

| 目前狀態 | 端點 | 結果狀態 | 其他狀態下呼叫 → |
|---------|------|---------|------------------|
| OPEN | POST /flashback/database（執行） | FLASHBACKED | 409, error_code=null, detail 含目前狀態 |
| FLASHBACKED | POST /flashback/database/finalize（執行） | OPEN | 409 |

FLASHBACKED = 已回溯、READ ONLY 人工驗證窗。其餘所有 mutation 端點不改 db_state、
也不檢查 db_state（table/RP 操作在 FLASHBACKED 下技術上可呼叫——真實 Oracle 會因
read-only 失敗，本 API 不額外攔，風險接受並記錄於 §9）。

**狀態檢查的閘門歸屬（刻意不對稱，理由如下）**：
- `POST /flashback/database` 的 `db_state=OPEN` 檢查屬**閘門 5 前置條件**——
  dry_run 時以 checks 收集呈現（AC-DB-2 的 `db_state_open`）
- `POST /flashback/database/finalize` 的 `db_state=FLASHBACKED` 檢查屬**閘門 3
  資源解析**——「沒有進行中的 flashback」等於資源不存在，**dry_run 也直接 409**
  （AC-FZ-3），沒有 checks 可回（無對象可檢查）

## §3 各端點驗收準則（EARS）

### GET /health
- **AC-HL-1**: WHEN 呼叫 THE SYSTEM SHALL 回 200 `{"status": "ok"}`

### GET /flashback/status
- **AC-ST-1**: WHEN 呼叫 THE SYSTEM SHALL 回 200，body = DatabaseStatus 全欄位 ＋
  `fra_usage_percent: float`（`fra_used/fra_limit*100` 四捨五入 1 位）＋
  `preconditions: {"P1_archivelog": bool, "P2_flashback_on": bool, "P3_fra_space": bool}`
  （P3 = `fra_usage_percent < FRA_USAGE_THRESHOLD`，**嚴格小於**）

```json
{"log_mode": "ARCHIVELOG", "flashback_on": true, "db_state": "OPEN",
 "current_scn": 2000000, "oldest_flashback_scn": 1500000,
 "oldest_flashback_time": "2026-06-11T09:00:00", "retention_minutes": 1440,
 "fra_limit_bytes": 10737418240, "fra_used_bytes": 4294967296,
 "fra_usage_percent": 40.0, "estimated_flashback_size": 2147483648,
 "preconditions": {"P1_archivelog": true, "P2_flashback_on": true, "P3_fra_space": true}}
```

### GET /restore_points
- **AC-RPL-1**: WHEN 呼叫 THE SYSTEM SHALL 回 200
  `{"restore_points": [RestorePoint...]}`，scn 升冪；無資料回空陣列（200，非 404）

### POST /restore_points
Request：`{"name": str（必填 1–128 字）, "guarantee": bool = true, "dry_run": bool = true}`

- **AC-RPC-1**(happy): WHEN dry_run=false 且檢查全過 THE SYSTEM SHALL 建立
  restore point（name 轉大寫；scn = 執行當下 current_scn）並回 200
  `{"dry_run": false, "restore_point": RestorePoint}`
- **AC-RPC-2**(edge): WHEN dry_run=true THE SYSTEM SHALL 回 200
  `{"dry_run": true, "would_create": "<大寫NAME>", "checks": {"P3_fra_space", "no_duplicate_name"}}`
  且不改變任何狀態
- **AC-RPC-3**(failure): WHEN dry_run=false 且同名（大小寫不敏感）已存在
  THE SYSTEM SHALL 回 409 error_code=ORA-38796
- **AC-RPC-4**(failure): WHEN dry_run=false 且 FRA 使用率 ≥ 門檻
  THE SYSTEM SHALL 回 409 error_code=ORA-19809
- **AC-RPC-5**(edge): WHEN guarantee 省略 THE SYSTEM SHALL 視為 true

### DELETE /restore_points/{name}
Query：`dry_run: bool = true`

- **AC-RPD-1**(happy): WHEN dry_run=false 且存在 THE SYSTEM SHALL 刪除並回 200
  `{"dry_run": false, "dropped": RestorePoint}`（含完整資料供留檔）
- **AC-RPD-2**(edge): WHEN dry_run=true 且存在 THE SYSTEM SHALL 回
  `{"dry_run": true, "would_drop": RestorePoint}` 且不刪除
- **AC-RPD-3**(failure): WHEN 不存在（含 dry_run=true）THE SYSTEM SHALL 回 404
  error_code=ORA-38780（閘門 3 資源解析先於一切）

### GET /recyclebin
- **AC-RB-1**: WHEN 呼叫 THE SYSTEM SHALL 回 200 `{"entries": [RecycleBinEntry...]}`
  droptime 降冪
- **AC-RB-2**(edge): WHEN `owner` 省略 THE SYSTEM SHALL 回全部；給定時大小寫不敏感過濾

### POST /flashback/table
Request：
```json
{"owner": "SCOTT", "table_name": "EMP",
 "target": {"scn": 1700000} | {"timestamp": "2026-06-12T09:00:00"},
 "enable_row_movement": false, "dry_run": true}
```

**target 解析（canonical = SCN），全 spec 統一三步序**：
1. **P4 邊界檢查作用在「原始輸入形式」**：scn 輸入比對 `oldest_flashback_scn`、
   timestamp 輸入比對 `oldest_flashback_time`（字串轉 datetime 比較）
2. P4 通過後，timestamp 才由 repository `timestamp_to_scn(ts)` 解析成 SCN
   （mock 公式見 §4，以 current_scn 封頂，故解析值永不為未來點）
3. 之後一律以解析後 SCN 下達
- `target` 必須**恰好一個** key，否則 422（pydantic，閘門 2）
- 未來點檢查（AC-FT-6 的 422）**只作用於 raw scn 輸入**——timestamp 經封頂解析
  不可能觸發

- **AC-FT-1**(happy): WHEN dry_run=false、目標表存在、P1/P4/P6 全過
  THE SYSTEM SHALL 執行回溯並回 200
  `{"dry_run": false, "flashed_back": "OWNER.TABLE", "prior_scn": <執行前 current_scn>,
    "executed_scn": <實際下達的 SCN>, "note": <回退提示文案>}`
- **AC-FT-2**(happy): WHEN target 用 timestamp 且通過檢查 THE SYSTEM SHALL
  先呼叫 `timestamp_to_scn` 取得 SCN，`executed_scn` = 該解析值（**不是** current_scn）
- **AC-FT-3**(edge): WHEN dry_run=true THE SYSTEM SHALL 回
  `{"dry_run": true, "prior_scn": <目前 current_scn>, "checks": {"P1_archivelog",
   "P4_within_retention", "P6_row_movement"}}` 且不改狀態（prior_scn 在 dry_run
  也給——caller 先記回退點再執行）；dry_run 下 `enable_row_movement=true` **不會**
  實際執行 ALTER，checks 反映表的「目前」狀態
- **AC-FT-4**(failure): WHEN 表不存在 THE SYSTEM SHALL 回 404 error_code=null
  （dry_run 也 404，閘門 3）
- **AC-FT-5**(failure): WHEN target.scn **<** oldest_flashback_scn（嚴格小於；等於可過）
  或 target.timestamp **<** oldest_flashback_time THE SYSTEM SHALL（dry_run=false 時）
  回 409 error_code=ORA-38729
- **AC-FT-6**(failure): WHEN target.scn **>** current_scn THE SYSTEM SHALL 回 422
  error_code=null（未來點不是領域狀態問題，是輸入無效；timestamp 解析值
  由 `timestamp_to_scn` 以 current_scn 封頂，不會觸發本條）
- **AC-FT-7**(failure): WHEN P6 row_movement=false 且 enable_row_movement=false
  THE SYSTEM SHALL（dry_run=false 時）回 409 error_code=ORA-08189
- **AC-FT-8**(edge): WHEN P6 違反且 enable_row_movement=true 且 dry_run=false
  THE SYSTEM SHALL 先執行 ENABLE ROW MOVEMENT 再回溯（成功路徑同 AC-FT-1）
- **AC-FT-9**(edge): WHEN 同一表連續成功回溯兩次 THE SYSTEM SHALL 兩次都執行
  （無冪等防護），各自留 audit

### POST /flashback/drop
Request：`{"owner": str, "table_name": str（original_name）, "rename_to": str|null = null, "dry_run": true}`

- **AC-FD-1**(happy): WHEN dry_run=false、recyclebin 有該表、無名稱衝突
  THE SYSTEM SHALL 還原 droptime 最近一筆並回 200
  `{"dry_run": false, "restored_as": "<大寫名>", "from_entry": RecycleBinEntry, "note": <BIN$ 索引提示>}`
- **AC-FD-2**(edge): WHEN rename_to=null THE SYSTEM SHALL 還原為 original_name
- **AC-FD-3**(edge): WHEN dry_run=true THE SYSTEM SHALL 回
  `{"dry_run": true, "would_restore": RecycleBinEntry, "restored_as": str}` 不改狀態
- **AC-FD-4**(failure): WHEN recyclebin 無該表（含 dry_run）THE SYSTEM SHALL 回 404
  error_code=ORA-38305
- **AC-FD-5**(failure): WHEN 還原目標名（rename_to 或 original_name）已存在於現有表
  THE SYSTEM SHALL 回 409 error_code=ORA-38312（dry_run 時以 checks 呈現，不擲出）

### POST /flashback/database
Request：
```json
{"target": {"restore_point": "NAME"} | {"scn": 1800000} | {"timestamp": "2026-06-11T22:00:00"},
 "dry_run": true, "confirm": null, "approval_id": null}
```
target 恰好一個 key（422）。**target 解析順序（閘門 3）**：
restore_point → 查名單，不存在 404 ORA-38780（dry_run 也 404），存在則取其 scn；
timestamp → P4 邊界檢查後由 `timestamp_to_scn` 解析；scn → 直接用。

- **AC-DB-1**(happy): WHEN dry_run=false、confirm/approval_id 齊備、P1–P4 過、
  db_state=OPEN THE SYSTEM SHALL 執行（SHUTDOWN→MOUNT→FLASHBACK→OPEN READ ONLY），
  結束 db_state=FLASHBACKED、current_scn=目標 SCN，回 200
  `{"dry_run": false, "db_state": "FLASHBACKED", "flashed_back_to_scn": <目標 SCN>,
    "next_step": <finalize 提示>}`（三種 target 形式 flashed_back_to_scn 都必為
  解析後的實際 SCN）
- **AC-DB-2**(edge): WHEN dry_run=true THE SYSTEM SHALL 回
  `{"dry_run": true, "checks": {"P1_archivelog","P2_flashback_on","P3_fra_space",
   "P4_within_retention","db_state_open"}, "estimated_flashback_size": int,
   "resolved_target_scn": int|null}` 不改狀態、不檢查 confirm。
  `estimated_flashback_size` 直接取 DatabaseStatus 同名欄位；
  `resolved_target_scn`：restore_point/scn 直接給值；timestamp 若 P4 檢查通過
  則解析給值（解析是唯讀操作），P4 不通過給 null
- **AC-DB-3**(failure): WHEN dry_run=false 且 confirm ≠ token 或 approval_id 空白
  THE SYSTEM SHALL 回 428 error_code=null（在前置條件檢查之前，閘門 4 先於 5）
- **AC-DB-4**(failure): WHEN target.restore_point 不存在 THE SYSTEM SHALL 回 404
  ORA-38780（先於 428——閘門 3 先於 4）
- **AC-DB-5**(failure): WHEN P2 flashback_on=false THE SYSTEM SHALL 回 409
  error_code=null，detail 以 "P2 violated" 開頭（P1/P3/P4 同理：P1 null、
  P3 ORA-19809、P4 ORA-38729）
- **AC-DB-6**(failure): WHEN db_state ≠ OPEN THE SYSTEM SHALL 回 409 error_code=null
  （§2.5 狀態機）

### POST /flashback/database/finalize
Request：`{"dry_run": true, "confirm": null, "approval_id": null}`

- **AC-FZ-1**(happy): WHEN dry_run=false、confirm/approval_id 齊備、db_state=FLASHBACKED
  THE SYSTEM SHALL 執行 OPEN RESETLOGS，db_state=OPEN，回 200
  `{"dry_run": false, "db_state": "OPEN", "warning": <RMAN 全備提示>}`
- **AC-FZ-2**(edge): WHEN dry_run=true 且 db_state=FLASHBACKED THE SYSTEM SHALL 回
  `{"dry_run": true, "would_finalize_at_scn": <dry_run 當下的 current_scn，
  非預測執行後值>}`，confirm 不檢查
- **AC-FZ-3**(failure): WHEN db_state ≠ FLASHBACKED（含 dry_run）THE SYSTEM SHALL
  回 409 error_code=null（狀態檢查屬閘門 3 資源解析性質：沒有可 finalize 的對象）
- **AC-FZ-4**(failure): WHEN dry_run=false 且 confirm/approval_id 缺 THE SYSTEM SHALL
  回 428

### GET /audit/log
- **AC-AU-1**: WHEN 呼叫 THE SYSTEM SHALL 回 200 `{"entries": [AuditEntry...]}`
  時間降冪；`limit` 預設 100，夾擠至 [1, 1000]

### auth（全 mutation 端點）
- **AC-AUTH-1**: WHEN `FLASHBACK_API_KEY` 已設且 mutation 請求無/錯 key
  THE SYSTEM SHALL 回 401（閘門 1，先於 422/404/428/409）
- **AC-AUTH-2**: WHEN `FLASHBACK_API_KEY` 未設 THE SYSTEM SHALL 不驗（dev 模式）；
  read 端點永不驗

**OracleRepository 介面**（docstring = real 模式照抄執行的指令）：

| 方法 | 對應 SQL/操作 |
|------|--------------|
| `get_status()` | `v$database`、`v$recovery_file_dest`、`v$flashback_database_log`、retention 參數 |
| `timestamp_to_scn(ts: str) -> int` | `SELECT TIMESTAMP_TO_SCN(TO_TIMESTAMP(:ts,'YYYY-MM-DD"T"HH24:MI:SS')) FROM dual` |
| `list_restore_points()` | `SELECT name, scn, time, guarantee_flashback_database, storage_size FROM v$restore_point ORDER BY scn` |
| `create_restore_point(name, guarantee)` | `CREATE RESTORE POINT <name> [GUARANTEE FLASHBACK DATABASE]` |
| `drop_restore_point(name)` | `DROP RESTORE POINT <name>` |
| `list_recyclebin(owner)` | `SELECT ... FROM dba_recyclebin [WHERE owner=:o] ORDER BY droptime DESC` |
| `get_table(owner, table)` | `SELECT row_movement FROM dba_tables WHERE owner=:o AND table_name=:t` |
| `enable_row_movement(owner, table)` | `ALTER TABLE <o>.<t> ENABLE ROW MOVEMENT` |
| `flashback_table(owner, table, scn)` | `FLASHBACK TABLE <o>.<t> TO SCN <scn>` |
| `flashback_drop(owner, table, rename_to)` | `FLASHBACK TABLE <o>.<t> TO BEFORE DROP [RENAME TO <r>]`，還原 droptime 最近一筆，回 `{"restored_as", "from_entry"}` |
| `flashback_database(scn)` | `SHUTDOWN IMMEDIATE; STARTUP MOUNT; FLASHBACK DATABASE TO SCN <scn>; ALTER DATABASE OPEN READ ONLY` → db_state=FLASHBACKED、current_scn=scn |
| `open_resetlogs()` | `SHUTDOWN IMMEDIATE; STARTUP MOUNT; ALTER DATABASE OPEN RESETLOGS` → db_state=OPEN |
| `append_audit(entry)` / `list_audit(limit)` | real：audit 表；mock：in-memory list |

**Mock 初始狀態與模擬規則**（`MOCK_ORACLE=true`；測試與文件範例都用這份）：

```python
log_mode="ARCHIVELOG", flashback_on=True, db_state="OPEN",
current_scn=2_000_000,
oldest_flashback_scn=1_500_000, oldest_flashback_time="2026-06-11T09:00:00",
retention_minutes=1440, fra_limit_bytes=10*2**30, fra_used_bytes=4*2**30,
estimated_flashback_size=2*2**30,
restore_points={"BEFORE_UPGRADE_20260611":
    {name, scn=1_800_000, time="2026-06-11T22:00:00", guarantee=True, storage_size=1*2**30}},
tables={("SCOTT","EMP"): row_movement=True, ("SCOTT","DEPT"): row_movement=False},
recyclebin=[{owner="SCOTT", object_name="BIN$jx8kQ3vT==$0",
             original_name="BONUS", droptime="2026-06-12T08:30:00"}],
audit=[]
```

模擬規則：
- 每次**寫入類**操作（create/drop RP、enable row movement、flashback table、
  flashback drop、open_resetlogs）`current_scn += 1`；
  `flashback_database(scn)` 例外：`current_scn = scn`（不 +1）
- guaranteed RP：建立 `fra_used_bytes += 1GiB`、刪除 `-= storage_size`
- **`timestamp_to_scn(ts)` mock 公式（確定性）**：
  `scn = min(oldest_flashback_scn + int((ts - oldest_flashback_time).total_seconds()), current_scn)`
  （從 oldest 基準每秒 +1 SCN，以 current_scn 封頂——所以 mock 下 timestamp
  解析值永不超過 current_scn，AC-FT-6 的 422 只有 raw scn 輸入會觸發）
- `flashback_drop`：從 recyclebin 移除該筆、`tables[(owner, restored_as)]` 新增
  （row_movement=False）

| error_code | 條件 | HTTP | detail 必含的處置建議 |
|-----------|------|------|----------------------|
| ORA-38729 | P4 超出保留範圍 | 409 | 改用 RMAN point-in-time recovery |
| ORA-08189 | P6 無 ROW MOVEMENT | 409 | 帶 enable_row_movement=true 重試 |
| ORA-38305 | 不在 recyclebin | 404 | 改用 RMAN 或輔助庫匯出 |
| ORA-19809 | P3 FRA 不足 | 409 | 加大 FRA、刪過期備份/已用 GRP |

| ORA-38796 | restore point 同名 | 409 | 改名或先 DELETE |

| null | P1/P2 違反、狀態機 409、428、404 表不存在、422 | 各 AC | detail 以 "Pn violated:" 或狀態說明開頭 |

## §7 審計（AuditEntry）

**寫入時機：每個 mutation request 恰好一筆**，無論結果（401/422 除外——閘門 1/2
在進 service 前就擋掉，不留審計；閘門 3 之後全留）。

```json
{"timestamp": "ISO8601", "operator": "<API key 前 8 碼（不足 8 碼取全部）或 'dev'>",
 "operation": "create_restore_point|drop_restore_point|flashback_table|flashback_drop|flashback_database|finalize_database",
 "target": "SCOTT.EMP"|"<RP名>"|null, "target_scn": int|null, "target_time": str|null,
 "dry_run": bool, "approval_id": str|null,
 "result": "success" | "dry_run" | "rejected:<error_code 或短原因>" | "error:<msg>"}
```

result 模式枚舉封閉：四種，不得發明第五種。`error:` 保留給非 FlashbackError 的
意外例外（service 最外層 catch）。

**各 operation 的欄位對應**：

| operation | target | target_scn | target_time |
|-----------|--------|-----------|-------------|
| create/drop_restore_point | RP 名（大寫） | null | null |
| flashback_table | "OWNER.TABLE" | 原始 scn 輸入（timestamp 輸入時 null） | 原始 timestamp 輸入（scn 輸入時 null） |
| flashback_drop | "OWNER.TABLE" | null | null |
| flashback_database | restore_point 名或 null | 原始/解析 scn 或 null | 原始 timestamp 或 null |
| finalize_database | null | null | null |

## §8 測試計畫

每條 AC ≥ 1 測試，**測試函式名含 AC 編號**（如 `test_ac_ft_2_timestamp_resolves_scn`）。
另加 mock 狀態操縱類：FRA 撐爆（AC-RPC-4）、recyclebin 加同名衝突項（AC-FD-5）、
完整 database flashback→finalize 流（AC-DB-1→AC-FZ-1 串連）。
audit 驗證：dry_run / rejected / success 三種 result 各至少出現一次斷言。

## §9 Out of Scope

| 項目 | 原因 | 替代支援 |
|------|------|---------|
| 202+job 非同步模式 | 單實例管理 API，一期從簡 | §0.5 註明 client timeout 責任 |
| FLASHBACKED 狀態下擋 table/RP 操作 | 真實 Oracle 自會以 read-only 拒絕 | §2.5 風險註記 |
| 業務資料人工驗證（SOP §4.1-4/§5-4） | 需人工 | FLASHBACKED 驗證窗＋prior_scn 回退 |
| 應用停機通知（SOP §5-2） | 組織流程 | response 文案 |
| RMAN 全備（SOP §5-5） | 屬備份 SOP | finalize warning |
| BIN$ 索引 rename（SOP §4.2-3） | 逐案命名 | response note |
| recyclebin 參數修改（P5） | 實例層設定 | 404 detail 說明 |
