# MinIO Bucket Disaster Recovery API Spec

> Generated from: `sop/minio-bucket-disaster-recovery.md`（OPS-SOP-021）
> Generated by: `.claude/skills/sop-to-spec`（v3）
> 本文件自足：實作 agent 只讀本文件。未定義行為 = spec bug，回報而非自行發明。

服務：`bucket-dr-api`，port **8004**，FastAPI，三層式（api / service / repository）。

## §0 全域規則

### 0.1 閘門順序

```
1. auth        → 401（mutation 端點缺/錯 X-API-Key；read 端點不驗）
2. schema 驗證 → 422（pydantic：必填、型別）
3. 資源解析    → 404（bucket / object key / version id 不存在；dry_run 也 404）
4. 風險閘門    → 428（irreversible 且 dry_run=false：confirm/approval_id 缺漏）
5. 前置條件    → 409（領域狀態不允許）
6. 執行
dry_run=true：跑 1–3（404 照常擲出），**接著執行閘門 5 的全部檢查但以
「收集不擲出」模式**填進 checks；閘門 4 與 6 完全跳過
```

### 0.2 統一 response 形狀

```json
// dry_run=true：
{"dry_run": true,
 "checks": {"<條件名>": {"ok": true}
            或 {"ok": false, "detail": "...", "error_code": "<錯誤名>"|null}},
 ...端點唯讀附加欄位}
// dry_run=false 成功：{"dry_run": false, ...端點專屬欄位}
// 錯誤：{"detail": "...，前置條件違反以 'Cn violated: ' 開頭", "error_code": "<錯誤名>"|null}
```

### 0.2.1 全域型別約定

- 時間欄位一律 ISO8601 秒精度 naive UTC：`YYYY-MM-DDTHH:MM:SS`
- bool 欄位永遠出現
- 「最新版本」= versions 清單中 `last_modified` 最大者；平手以插入順序（FIFO）後者為新
- object key 含 `/`，一律放 **query/body 參數**，不放 path

### 0.3 風險閘門常數

- confirm token：`"I-UNDERSTAND-PERMANENT-DELETION"`——唯一出處
  `models/schemas.py::CONFIRM_TOKEN`，service 與測試一律 import
- `approval_id`：strip 後長度 > 0；confirm/approval_id 驗證發生在閘門 4
  （service 層，僅 irreversible 且 dry_run=false），schema 上皆 `Optional[str] = None`

### 0.4 auth、DI、錯誤模式（inline，不依賴其他檔案）

- auth：`X-API-Key` ↔ env `BUCKET_DR_API_KEY`，`secrets.compare_digest`，
  **每 request 讀 env**；未設 = dev 模式不驗；read 端點永不驗
- DI：`functools.lru_cache(maxsize=1)` providers（`get_minio()`/`get_service()`）+
  FastAPI `Depends`；`reset_singletons()` 供測試；`main.py` lifespan 呼叫一次
  `get_service()` warm-up（real 模式缺 `MINIO_ENDPOINT` → boot fail-hard）
- service 擲 `BucketDRError(status_code, detail, error_code)`；app 級 handler 轉
  §0.2 錯誤 body
- 固定文案集中 `models/schemas.py` 常數（§7.1）

### 0.5 同步模型與並發

- 全端點同步（單物件操作皆毫秒級）
- 並發：無防護（last-write-wins）；單人維運操作假設，風險接受
- 冪等：restore_version 重打會再疊一個新版本（AC-RV-4）；undelete 第二次因
  delete marker 已不在最新而 409（AC-UD-3）；purge 第二次 404（版本已不存在）

## §1 Domain Model

| 實體 | 欄位 |
|------|------|
| BucketInfo | `bucket: str`、`versioning: str`（"Enabled"/"Suspended"/"Unversioned"） |
| ObjectVersion | `version_id: str`、`is_delete_marker: bool`、`is_latest: bool`、`etag: str|null`（delete marker 為 null）、`size: int|null`（同左）、`last_modified: str` |
| AuditEntry | §7 |

無狀態機：本服務不改任何實體的多值狀態（versioning 狀態唯讀，不提供修改——SOP §6
的 `mc version enable` 屬防再發處置，列 Out of Scope）。

## §3 各端點驗收準則（EARS）

### GET /health
- **AC-HL-1**: WHEN 呼叫 THE SYSTEM SHALL 回 200 `{"status": "ok"}`

### GET /buckets/{bucket}/versioning
- **AC-VS-1**(happy): WHEN bucket 存在 THE SYSTEM SHALL 回 200
  `{"bucket": str, "versioning": "Enabled"|"Suspended"|"Unversioned",
    "preconditions": {"C2_versioning_enabled": bool}}`
- **AC-VS-2**(failure): WHEN bucket 不存在 THE SYSTEM SHALL 回 404
  error_code=`NoSuchBucket`

### GET /object/versions
Query：`bucket: str`（必填）、`key: str`（必填）

- **AC-VL-1**(happy): WHEN bucket 與 key 存在 THE SYSTEM SHALL 回 200
  `{"versions": [ObjectVersion...]}`，`last_modified` 降冪，
  delete markers 一併列出（`is_delete_marker: true`）
- **AC-VL-2**(failure): WHEN bucket 不存在 THE SYSTEM SHALL 回 404 `NoSuchBucket`
- **AC-VL-3**(failure): WHEN key 無任何版本 THE SYSTEM SHALL 回 404 `NoSuchKey`

### POST /object/restore_version（誤覆寫復原，SOP §3.2）
Request：
```json
{"bucket": "data-bucket", "key": "reports/q1.csv",
 "version_id": "v-0001", "dry_run": true}
```

- **AC-RV-1**(happy): WHEN dry_run=false、版本存在且非 delete marker、C2 過
  THE SYSTEM SHALL 以該版本內容建立**新的最新版本**（原版本鏈不動）並回 200
  `{"dry_run": false, "restored_from_version": str, "new_version": ObjectVersion,
    "note": <§7.1 NOTE_RV>}`
- **AC-RV-2**(edge): WHEN dry_run=true THE SYSTEM SHALL 回
  `{"dry_run": true, "checks": {"C2_versioning_enabled", "version_not_delete_marker"},
    "would_restore_from": ObjectVersion}` 且不改狀態
  （只回「來源版本」；**不預測** new_version——etag/last_modified 執行前未知）
- **AC-RV-3**(failure): WHEN bucket/key/version_id 不存在（含 dry_run）
  THE SYSTEM SHALL 回 404，error_code 依序 `NoSuchBucket`/`NoSuchKey`/`NoSuchVersion`
- **AC-RV-4**(edge): WHEN 同請求成功重打兩次 THE SYSTEM SHALL 兩次都執行，
  產生兩個新版本，各留 audit
- **AC-RV-5**(failure): WHEN version_id 指向 delete marker THE SYSTEM SHALL
  （dry_run=false 時）回 409 error_code=null，detail 說明 delete marker 不能作為
  還原來源（dry_run 時以 checks `version_not_delete_marker` 呈現）
- **AC-RV-6**(failure): WHEN bucket versioning ≠ "Enabled" THE SYSTEM SHALL
  （dry_run=false 時）回 409 error_code=`VersioningNotEnabled`，
  detail 以 "C2 violated: " 開頭並建議改走備份還原（SOP §6）

### POST /object/undelete（誤刪復原，SOP §3.3）
Request：`{"bucket": str, "key": str, "dry_run": true}`
（不收 version_id——目標固定為「最新版本且為 delete marker」者）

- **AC-UD-1**(happy): WHEN dry_run=false、key 的最新版本是 delete marker、C2 過
  THE SYSTEM SHALL 移除該 delete marker 並回 200
  `{"dry_run": false, "removed_delete_marker": str,
    "current_version": ObjectVersion, "note": <§7.1 NOTE_UD>}`
  （current_version = 移除後的**新最新版本，不爬鏈**——若它本身又是
  delete marker，照實回傳 `is_delete_marker: true`，代表物件仍處刪除狀態，
  caller 可再呼叫一次 undelete）
- **AC-UD-2**(edge): WHEN dry_run=true THE SYSTEM SHALL 回
  `{"dry_run": true, "checks": {"C2_versioning_enabled", "latest_is_delete_marker"},
    "would_remove": ObjectVersion|null}` 不改狀態（latest 非 delete marker 時
  would_remove=null）
- **AC-UD-3**(failure): WHEN key 的最新版本**不是** delete marker THE SYSTEM SHALL
  （dry_run=false 時）回 409 error_code=`DeleteMarkerNotLatest`，detail 說明
  物件目前未被刪（SOP §6 第 4 列）
- **AC-UD-4**(failure): WHEN bucket/key 不存在 THE SYSTEM SHALL 回 404
  （`NoSuchBucket`/`NoSuchKey`）
- **AC-UD-5**(failure): WHEN versioning ≠ Enabled THE SYSTEM SHALL
  （dry_run=false 時）回 409 `VersioningNotEnabled`

### POST /object/purge_versions（**irreversible**，SOP §4）
Request：
```json
{"bucket": str, "key": str, "dry_run": true, "confirm": null, "approval_id": null}
```

- **AC-PG-1**(happy): WHEN dry_run=false、confirm/approval_id 齊備、key 存在
  THE SYSTEM SHALL 刪除該 key 全部版本與 delete markers 並回 200
  `{"dry_run": false, "purged_versions": int, "warning": <§7.1 WARNING_PG>}`
  （purged_versions = 實體版本＋delete markers 合計）
- **AC-PG-2**(edge): WHEN dry_run=true THE SYSTEM SHALL 回
  `{"dry_run": true, "checks": {}, "would_purge_versions": int}` 不改狀態、
  不檢查 confirm（本端點無 409 級前置條件，checks 為空物件——形狀仍保留）
- **AC-PG-3**(failure): WHEN dry_run=false 且 confirm ≠ token 或 approval_id 空白
  THE SYSTEM SHALL 回 428 error_code=null
- **AC-PG-4**(failure): WHEN bucket/key 不存在（含 dry_run）THE SYSTEM SHALL 回 404
  （`NoSuchBucket`/`NoSuchKey`；404 先於 428——閘門 3 先於 4）
- **AC-PG-5**(edge): WHEN purge 後同 key 再 purge THE SYSTEM SHALL 回 404 `NoSuchKey`
- **AC-PG-6**(edge): WHEN versioning=Suspended THE SYSTEM SHALL 照常執行
  （purge 不依賴 C2——刪除不需要版本機制健在；SOP §4 未列 C2 為前置）

### GET /audit/log
- **AC-AU-1**: WHEN 呼叫 THE SYSTEM SHALL 回 200 `{"entries": [AuditEntry...]}`
  時間降冪；limit 預設 100，夾擠 [1, 1000]

### auth
- **AC-AUTH-1**: WHEN `BUCKET_DR_API_KEY` 已設且 mutation 無/錯 key
  THE SYSTEM SHALL 回 401（先於 422/404/428/409）
- **AC-AUTH-2**: WHEN 未設 THE SYSTEM SHALL 不驗；read 端點永不驗

## §4 三層落點

```
bucket-dr-api/
  main.py                       create_app() + BucketDRError handler + lifespan warm-up
  core/config.py                Settings（MOCK_MINIO、MINIO_ENDPOINT/ACCESS/SECRET）
  core/deps.py                  get_minio()/get_service() + reset_singletons()
  api/dependencies.py           require_api_key、operator_from_key（key 前 8 碼或 'dev'）
  api/routers/system.py         /health /audit/log
  api/routers/objects.py        /buckets/{b}/versioning /object/*
  services/recovery_service.py  閘門 3–6、Cn 檢查、audit
  models/schemas.py             pydantic + CONFIRM_TOKEN + §7.1 文案常數
  repository/minio_client.py    MinioRepository ABC + RealMinioRepository（minio SDK）
  repository/mock_minio.py      MockMinioRepository（MOCK_MINIO=true）
  tests/  pytest.ini（testpaths = tests）  requirements.txt  Dockerfile（port 8004）
```

**MinioRepository 介面**（docstring = real 模式對應 SDK/mc 操作）：

| 方法 | 對應操作 |
|------|---------|
| `get_bucket(bucket) -> BucketInfo|None` | `mc version info <alias>/<bucket>`（SDK：`get_bucket_versioning`） |
| `list_versions(bucket, key) -> list[ObjectVersion]` | `mc ls --versions`（SDK：`list_objects(versions=True, prefix=key)`），不存在回空 list。**repository 永不擲業務錯誤**：空值/None 回傳，404 判定一律在 service（空 list → NoSuchKey） |
| `restore_version(bucket, key, version_id) -> ObjectVersion` | `mc cp --version-id <vid> <src> <src>`（SDK：copy_object 自身，回新版本） |
| `remove_version(bucket, key, version_id) -> None` | `mc rm --version-id <vid>`（undelete=移除 delete marker；purge 逐版本呼叫） |
| `append_audit(entry)` / `list_audit(limit)` | real：audit bucket/table；mock：in-memory |

**Mock 初始狀態與模擬規則**（`MOCK_MINIO=true`）：

```python
buckets = {"data-bucket": "Enabled", "legacy-bucket": "Suspended"}
objects = {
  ("data-bucket", "reports/q1.csv"): [   # last_modified 升冪存放
    {version_id: "v-0001", is_delete_marker: False, etag: "etag-a",
     size: 1024, last_modified: "2026-06-10T10:00:00"},
    {version_id: "v-0002", is_delete_marker: False, etag: "etag-b",
     size: 2048, last_modified: "2026-06-11T10:00:00"},   # latest
  ],
  ("data-bucket", "archive/old.log"): [
    {version_id: "v-0010", is_delete_marker: False, etag: "etag-c",
     size: 512, last_modified: "2026-06-09T08:00:00"},
    {version_id: "dm-0011", is_delete_marker: True, etag: None,
     size: None, last_modified: "2026-06-12T07:00:00"},   # latest = 被刪狀態
  ],
  ("legacy-bucket", "dump.bin"): [
    {version_id: "v-0100", is_delete_marker: False, etag: "etag-d",
     size: 4096, last_modified: "2026-06-08T12:00:00"},
  ],
}
audit = []
_version_seq = 1000   # mock 產生新 version_id："v-1001", "v-1002", ...
                      # reset_singletons() 重建 repo → seq 回到 1000
```

模擬規則：`restore_version` 複製來源版本 etag/size、`version_id 取 "v-"+next(seq)`、
`last_modified = now()`、append 為最新；`remove_version` 從 list 移除該筆；
`is_latest` 由 repository 回傳時計算（list 末筆 = latest），不存欄位。

## §5 設定（環境變數）

| 變數 | 預設 | 範圍 | 讀取時機 |
|------|------|------|---------|
| `MOCK_MINIO` | `false` | true/false | boot（Settings 快取） |
| `MINIO_ENDPOINT`/`MINIO_ACCESS_KEY`/`MINIO_SECRET_KEY` | — | real 必填 | boot；缺 → warm-up fail-hard |
| `BUCKET_DR_API_KEY` | 空 | — | **每 request** |

## §6 錯誤模型

| error_code | 條件 | HTTP | detail 處置建議 |
|-----------|------|------|----------------|
| NoSuchBucket | bucket 不存在（C1） | 404 | 確認 bucket 名 |
| NoSuchKey | key 無任何版本 | 404 | `GET /object/versions` 確認 |
| NoSuchVersion | version_id 不存在（C3） | 404 | 重新查版本清單；無版本改走備份還原 |
| VersioningNotEnabled | C2 違反（restore/undelete） | 409 | 改走備份還原；防再發見 SOP §6 |
| DeleteMarkerNotLatest | undelete 目標非「目前被刪」 | 409 | 物件未被刪，確認 key |
| null | RV-5 delete marker 來源、428、422 | 各 AC | — |

## §7 審計（AuditEntry）

寫入時機：**每個 mutation request 恰好一筆**（401/422 在 service 前擋掉不留；
閘門 3 之後全留，含 dry_run 與被拒）。

```json
{"timestamp": "ISO8601", "operator": "<key 前 8 碼或 'dev'>",
 "operation": "restore_version|undelete|purge_versions",
 "bucket": str, "key": str, "version_id": str|null,
 "dry_run": bool, "approval_id": str|null,
 "result": "success" | "dry_run" | "rejected:<error_code 或短原因>" | "error:<msg>"}
```

`rejected:` 後綴規則：§6 有 error_code 用 error_code；error_code=null 的用
snake_case 短原因（如 `rejected:delete_marker_as_source`）。
`approval_id` 存 request 原值（strip 後）；未給或空白存 null。

| operation | version_id 欄位 |
|-----------|----------------|
| restore_version | 請求的 version_id |
| undelete | 被移除的 delete marker version_id（rejected 時 null） |
| purge_versions | null（目標是整條版本鏈） |

### §7.1 固定文案字面值

| 常數 | 字面值 |
|------|--------|
| NOTE_RV | `"原版本鏈不動，可再以舊 version_id 重複還原（SOP §3.2）"` |
| NOTE_UD | `"物件已回到上一個實體版本，請以 etag/size 驗證內容（SOP §5）"` |
| WARNING_PG | `"全部版本已永久刪除，無法復原（SOP §4）"` |

## §8 測試計畫

每條 AC ≥ 1 測試，測試名含 AC 編號。conftest：`MOCK_MINIO=true` ＋ autouse
`reset_singletons()`。mock 操縱類：versioning Suspended 路徑（AC-RV-6/UD-5 用
legacy-bucket）、delete marker 場景（archive/old.log）、purge 後再操作（AC-PG-5）。
audit：dry_run / rejected / success 三種 result 各至少一次斷言。

## §9 Out of Scope

| 項目 | 原因 | 替代支援 |
|------|------|---------|
| `mc version enable`（修改 versioning） | 實例層設定變更，防再發處置 | AC-RV-6 detail 指引 |
| bucket 層級整批還原 | SOP 僅涵蓋單物件 | 逐 key 呼叫 |
| lifecycle 規則管理 | 屬空間管理 SOP | NoSuchVersion detail 提示 |
| 磁碟/erasure set 損毀、replication failover | SOP §1 明示不涵蓋 | — |
| 業務內容驗證（SOP §5） | 需人工 | AC-UD-1/RV-1 回 etag/size 供比對 |
