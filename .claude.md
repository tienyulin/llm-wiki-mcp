# Claude 工作流程指南

## Git 工作流程（重要 ⭐）

每次開始做事都要遵循以下流程：

### 1. 準備
```bash
git fetch origin
git checkout main
git pull origin main
```

### 2. 建立 Feature Branch
```bash
git checkout -b feature/描述
# 或
git checkout -b fix/描述
```

### 3. 做改動和測試
- 在 feature branch 上完成所有工作
- 執行測試確保全部通過
- 提交變更

### 4. 推送和開 PR
```bash
git push -u origin feature/描述
```
然後開 PR 從 feature branch 合併回 main

### 5. PR 合併
等待審核和批准後合併到 main

## 注意事項

- ⛔ 不要直接推送到 main（受保護分支）
- ✅ 所有改動都要通過 PR
- ✅ 確保測試全部通過再開 PR
- ✅ PR 應該有清楚的標題和描述

## 命名規範

- Feature: `feature/add-xxx`
- Bug Fix: `fix/issue-xxx`
- Refactor: `refactor/xxx`
- Docs: `docs/xxx`
